Google Integration
Setup Instructions
  1. Go to the Google Cloud Console — Credentials
  2. Create a new project (or select an existing one)
  3. Click Create Credentials → OAuth client ID
  4. Set Application type to Web application
  5. Under Authorised redirect URIs, add:
    {{ url_for('google.callback', _external=True) }}
  6. Copy the Client ID and Client Secret into the fields below
  7. Enable the APIs you need in API Library :
    • Google Calendar API — for calendar sync and Meet links
    • Google Docs API — for document creation
    • Google Drive API — for file storage
  8. If your app is in Testing mode, add user emails under the OAuth consent screen
For production use, submit your app for Google verification to remove the "unverified app" warning.
Common Issues
  • Redirect URI mismatch — Ensure the Authorised redirect URI in Google Cloud Console matches exactly: {{ url_for('google.callback', _external=True) }}
  • Access denied — If your app is in Testing mode, check that each user's email is added under the OAuth consent screen's test users
  • Scopes not working — After changing scopes, users may need to disconnect and reconnect their Google account for the new scopes to take effect
